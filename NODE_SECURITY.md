# Node.js 보안 가이드 (TEND+ 개발 환경)

> **개발 전 Node.js 버전 확인 필수**  
> 아래 권장 버전 미만 사용 시 보안 취약점에 노출될 수 있습니다.

---

## 🛡️ 권장 버전 (2026년 기준)

| 라인 | 최소 버전 | 비고 |
|------|----------|------|
| **v22.x (LTS)** | v22.22.0 이상 | 권장 |
| **v20.x (Maintenance LTS)** | v20.20.0 이상 | 지원 중 |

```bash
# 버전 확인
node --version
# v22.22.0 이상 또는 v20.20.0 이상이어야 함
```

---

## 🚨 2026년 1월 주요 보안 릴리스

**영향받는 버전 (패치 필요):**
- 25.x: v25.3.0 미만
- 24.x: v24.13.0 미만
- 22.x: v22.22.0 미만
- 20.x: v20.20.0 미만

**주요 취약점:**
- **CVE-2025-55130 (High):** 파일 시스템 권한 모델 우회 → 허용되지 않은 경로 파일 읽기/쓰기
- **CVE-2025-55131 (High):** 초기화되지 않은 메모리 노출
- **CVE-2025-59465 (High):** 에러 핸들링 오류로 인한 DoS (복구 불가 크래시)

---

## 🟠 2025년 주요 보안 이슈

- **CVE-2025-27209:** HashDoS (해시 충돌 이용)
- **CVE-2025-27210:** 경로 탐색 우회 (장치명 처리 미흡)

---

## 🟡 Windows 전용 취약점 (2024년)

**CVE-2024-36138 (High):** 배치 파일(.bat, .cmd) 처리 시 임의 명령 실행 가능

**영향받는 버전:** v18.20.4, v20.15.1, v22.4.1 **미만**

→ **Windows 사용자는 반드시 위 버전 이상으로 업그레이드**

---

## ⚠️ EOL(지원 종료) 버전

v18, v19, v21 등은 **공식 보안 패치 미제공**입니다.  
새 취약점 발견 시 패치가 없으므로 **가급적 v20.20.0+ 또는 v22.22.0+로 업그레이드**하세요.

---

## 📌 TEND+ 프로젝트 요구사항

- **최소:** Node.js v20.20.0 이상 (Maintenance LTS)
- **권장:** Node.js v22.22.0 이상 (LTS)

**자동 검증:** `pnpm check-node-version` (prebuild 시 빌드 전 자동 실행)
